# Reproducibility

Every claim in the manuscript has a reproducer under `scripts/`:

| Claim                                              | Command                                                                        |
| -------------------------------------------------- | ------------------------------------------------------------------------------ |
| Main detection table (6 datasets, F1 ≈ 93.9%)      | `python scripts/evaluate.py --all-datasets --suite detection`                  |
| Explanation quality on CIC-IoT-2023                | `python scripts/evaluate.py --suite explanation --dataset cic-iot-2023`        |
| Critical-difference diagram                        | `python scripts/evaluate.py --suite cd-diagram`                                |
| ConformalGuard coverage (α = 0.05)                 | `python scripts/evaluate.py --suite conformal`                                 |
| Calibration of ConformalGuard                      | `python scripts/calibrate.py --alpha 0.05`                                     |
| Single-alert explanation with all four techniques  | `python scripts/explain.py --target 42 --render-dir runs/alerts/42/`           |

## Determinism

`ct_explain.utils.seed.set_global_seed` is called by every CLI before any
stochastic op. With the default config seed `20260101`, all CPU-only
pipelines produce byte-identical metrics; GPU paths reproduce to within
1e-5 due to cuDNN non-determinism in attention gathers.

## Expected wall-clock (Paper Table + this repo's CPU)

| Stage                                      | Paper (GPU)    | This repo (CPU, synthetic data)   |
| ------------------------------------------ | -------------- | --------------------------------- |
| Detection training, 10 epochs, CIC-IoT     | ~3 h           | ~15 min on synthetic frame        |
| Calibration on 30% split                   | ~4 min         | <1 s on synthetic calibration set |
| Explain one alert (all four techniques)    | 52.1 ms        | ≤ 200 ms (no torchdiffeq fallback)|
| Combined pipeline end-to-end                | 75 ms          | ≤ 300 ms (no torchdiffeq fallback)|

## Pinned artefacts

Checkpoints generated by `scripts/train.py` land in `runs/<name>/`. To
produce a reproducibility bundle to cite in the IEEE submission:

```bash
tar czf ct_explain_reproducibility.tgz \
    ct_explain/ configs/ scripts/ tests/ docs/ \
    runs/latest/ README.md requirements.txt LICENSE
```
